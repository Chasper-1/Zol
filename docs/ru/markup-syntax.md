# Синтаксис разметки zml

Zol использует язык разметки **zml** (Zol Markup Language) — лёгкую семантическую разметку со строгими правилами парности. Файлы имеют расширение `.zml`.

## Встроенное форматирование

| Маркер | Стиль | Пример | Результат |
|--------|-------|--------|-----------|
| `**текст**` | Жирный | `**важно**` | **важно** |
| `//текст//` | Курсив | `//акцент//` | *акцент* |
| `__текст__` | Подчёркнутый | `__подч__` | <u>подчёркнуто</u> |
| `~~текст~~` | Зачёркнутый | `~~удалено~~` | ~~удалено~~ |
| `''текст''` | Верхний индекс | `''2''` | ² |
| `,,текст,,` | Нижний индекс | `,,2,,` | ₂ |
| `` `текст` `` | Код | `` `fn()` `` | `fn()` |
| `==текст==` | Подсветка | `==важно==` | (жёлтый фон) |
| `!!текст!!` | Спойлер (строчный) | `!!секрет!!` | (скрыто) |
| `!!!текст!!!` | Спойлер (блок) | `!!!спойлер!!!` | (скрытый блок) |
| `++текст++` | Вставка | `++новое++` | (зелёный текст) |
| `--текст--` | Удаление | `--старое--` | (красный + зачёркнутый) |
| `/*текст*\` | Комментарий | `/*не видно*\` | (скрыто) |
| `$текст$` | Формула (строчная) | `$x+y$` | (моноширинный зелёный) |
| `$$текст$$` | Формула (центр) | `$$x^2$$` | (центрированный моноширинный) |

## Правила парсера

### 1. Строгая парность

Форматирование применяется **только** если найдена полная пара открывающего и закрывающего маркеров. Незакрытый маркер считается обычным текстом.

```
**жирный**        → ✅ жирный
**не жирный      → ❌ **не жирный (простой текст)
```

### 2. Пробелы у маркеров

После открывающего маркера не должно быть пробела. Перед закрывающим маркером не должно быть пробела.

```
**текст**         → ✅
** текст**        → ❌ (пробел после **)
**текст **        → ❌ (пробел перед **)
```

Пробелы *внутри* контента разрешены:
```
**жирный текст**  → ✅
```

Границы строк считаются корректными (пробел не требуется).

### 3. Непустой контент

Между маркерами должен быть хотя бы один непробельный символ.

```
****            → ❌ простой текст
////            → ❌ простой текст
```

### 4. Экранирование

Обратный слеш `\` перед любым символом отключает его специальное значение. Сам слеш не отображается.

```
\*\*текст\*\*     → **текст** (простой текст)
\\               → \
```

### 5. Приоритет маркеров

Если маркеры начинаются с одних и тех же символов, приоритет у более длинного:

```
$$x$$           → формула по центру ($$ побеждает)
$x$             → строчная формула
!!!текст!!!      → блок спойлер
!!текст!!        → строчный спойлер
```

Маркеры проверяются от самого длинного к самому короткому.

### 6. Вложенность

Маркеры могут быть вложены. Стек закрывается строго в обратном порядке (LIFO).

```
**жирный //и курсив//**         → ✅
//курсив внутри **жирного**//   → ✅
**a **b** c**                    → ✅ (двойной жирный)
```

Стили комбинируются: вложенный маркер добавляет свой стиль к родительскому.

### 7. Многострочные маркеры

Следующие маркеры работают через несколько строк:

| Маркер | Тип |
|--------|------|
| `/* ... *\` | Комментарий |
| `$$ ... $$` | Формула по центру |
| `!!! ... !!!` | Блок спойлер |

Обычные маркеры (`**`, `//`, `~~` и т.д.) закрываются в **пределах одной строки**.

### 8. Вариант комментария

zml использует `/*текст*\` (обратный слеш-звёздочка) вместо традиционного `*/`, чтобы избежать конфликтов с маркерами, содержащими `*`.

## Внутренняя реализация

Пайплайн парсера:

```
Сырой текст → token::tokenize() → Vec<Token>
           → parser::parse()   → MarkupDoc (AST)
           → segmenter::to_document_cache() → DocumentCache
```

- **Токенизатор**: Один проход, O(n). Производит `Token::Text`, `Token::Open(MarkupStyle)`, `Token::Close(MarkupStyle)`, `Token::Newline`.
- **Парсер**: Стековый, O(n). Строит `MarkupDoc` с вложенными `MarkupNode::Formatted`.
- **Сегментатор**: Преобразует AST в `DocumentCache` — вектор `MarkupCache` (один на строку), каждый содержит список `Segment` с байтовыми смещениями и битовой маской стиля.
