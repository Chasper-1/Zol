# Формат .rdb (Read Book)

> **Статус:** проектирование  
> **Версия:** 0.1 (черновик)

---

## Описание

**.rdb** — универсальный бинарный контейнер для текста с опциональной разметкой.
Не привязан к zoll: может хранить любой текст (zoll, md, html, plain) с сохранением
или без сохранения разметки.

**Ключевые возможности:**
- Двухуровневое сжатие (семантический словарь + zstd) — на 20–30% компактнее чистого zstd
- Указатели стилей и интерактивных элементов (термины, ссылки, спойлеры, формулы)
- Чтение без парсинга — достаточно десериализации
- Мгновенное открытие — токены уже готовы, парсер не нужен
- Экспорт в MD, HTML, PDF, TXT без перепарирования исходника
- Встраивание изображений и переводов (multi-language) в один файл

## Архитектура сжатия

```
┌─ Исходный текст ──────────────────────────┐
│  «Политический деятель сказал политический  │
│   лозунг политического деятеля»             │
└────────────────────────────────────────────┘
                      │
                      ▼
┌─ Уровень 1: Семантический словарь ───────┐
│  Словарь: {1: политический, 2: деятель,    │
│            3: сказал, 4: лозунг}           │
│  Текст: 1 2 3 1 4 1 2                     │
│  (длинные/частые слова → индекс)           │
└────────────────────────────────────────────┘
                      │
                      ▼
┌─ Уровень 2: zstd ────────────────────────┐
│  Байтовый поток индексов → zstd-блок       │
└────────────────────────────────────────────┘
```

Два уровня сжатия работают с **разными типами избыточности**:

| Уровень | Тип избыточности | Пример |
|---|---|---|
| Словарь | Лексическая | «политический» ×50 |
| zstd | Структурная | Повтор паттерна `1 2 3 1 4 1 2` |

Они не мешают, а дополняют друг друга.

## VarInt-кодирование индексов словаря

Индексы слов кодируются переменным количеством байт — чем чаще слово, тем короче индекс.

| Диапазон индексов | Байт (первый) | Вместимость |
|---|---|---|
| 0–127 | `0xxxxxxx` — 1 байт | 128 самых частых слов |
| 128–16 511 | `10xxxxxx xxxxxxxx` — 2 байта | ~16K слов |
| 16 512–2 097 151 | `110xxxxx xxxxxxxxx xxxxxxxx` — 3 байта | ~2M слов |

Короткие слова (1–4 буквы) не включаются в словарь — замена на индекс
не даёт выигрыша. Алгоритм компиляции оценивает «жирность» каждого слова
(частота × длина) и принимает решение о включении в словарь автоматически.

## Указатели (стили, интерактивность)

Разметка хранится отдельно от текста в виде указателей:

```
(строка: u32, токен_на_строке: u16, тип_указателя: u8, данные: ...)
```

**Типы указателей:**

| Тип | Назначение | Данные |
|---|---|---|
| `Bold` / `Italic` / и т.д. | Стили разметки | — |
| `Term` | Определение термина | ID термина в словаре |
| `Spoiler` | Скрытый текст | — |
| `Link` | Ссылка | URL или ID документа |
| `Footnote` | Сноска | ID сноски |
| `Formula` | Математическая формула | ID формулы |
| `Image` | Встроенное изображение | Offset + size в контейнере |

Указатели **не участвуют в семантическом сжатии** — они достаточно компактны
сами по себе. zstd, однако, доожмёт повторяющиеся последовательности указателей.

Благодаря указателям, для чтения .rdb **не нужен парсер zoll** — только десериализация.

## Структура файла

```
┌─────────────────────────────┐
│  Magic: "rdb\0" (4 байта)   │
├─────────────────────────────┤
│  Заголовок (метаданные)      │
│  - автор, название, дата     │
│  - язык (для multi-language) │
├─────────────────────────────┤
│  Словарь                     │
│  - список уникальных слов    │
│  - частотная статистика      │
├─────────────────────────────┤
│  Текст (индексы слов)        │
│  - сжатый zstd-поток         │
├─────────────────────────────┤
│  Указатели (опционально)     │
│  - стили, ссылки, термины    │
│  - сжатый zstd-поток         │
├─────────────────────────────┤
│  Ресурсы (опционально)       │
│  - изображения, шрифты       │
│  - переводы (multi-language) │
│  - сжатый zstd-поток         │
└─────────────────────────────┘
```

## Статистика компиляции

При сборке .rdb выводится отчёт. Позволяет автору видеть,
какие слова дают наибольший выигрыш при сжатии.

```
Сжатие .zoll → .rdb
━━━━━━━━━━━━━━━━━━━━━━━
Исходный .zoll:              1 240 000 B
Чистый текст без маркеров:     980 000 B
После словаря:                 690 000 B
После zstd:                    210 000 B

Самые жирные слова:
  политический    (×312)  9 984 B →  312 B   (-96%)
  торжественность (×48)   1 440 B →   48 B   (-96%)
  государственный (×211)  4 644 B →  211 B   (-95%)
  ...

Экономия: 83% относительно .zoll
          56% относительно голого zstd

Время компиляции: 127 ms
```

## Конвертация

Из .rdb экспортируется любой формат **без перепарирования исходника**:

```
       ┌──→ .zoll    (восстановить маркеры из указателей)
       │
.rdb ──┼──→ .md      (указатели → md-синтаксис)
       │
       ├──→ .html     (указатели → HTML-теги)
       │
       ├──→ .pdf      (прямой рендер в страницы)
       │
       └──→ .txt      (только текст, без разметки)
```

## Защита контента

Формат не шифрует данные, но делает их извлечение «на коленке» затруднительным:

- Текст хранится в виде индексов словаря, а не в читаемом виде
- Указатели стилей — бинарные смещения без документации
- Возможна обфускация (перемешивание секций) поверх структуры
- Спецификация формата может не публиковаться в открытом доступе

**Это не DRM.** Это «честная защита» — файл можно прочитать через официальную
читалку, но скопировать разметку или извлечь текст сторонним инструментом
достаточно сложно.
